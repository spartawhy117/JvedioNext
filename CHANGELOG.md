# 变更日志

本文件面向仓库首页与 public 分发仓库展示，正式版本记录与 [doc/CHANGELOG.md](./doc/CHANGELOG.md) 保持同步。

## [5.0.0] - 2026-03-30

## [5.0.1] - 2026-03-31

- 修复 Worker 鉴权接入后影片卡、合集卡、演员头像和详情页背景图无法显示的问题，并补上海报与合集封面回归验证。

## [5.1.0] - 2026-03-31

- 优化 HDD 大库扫描性能与可观测性，收口目录跳过判断和默认分页链路，并补充关键回归验证。

## [5.2.0] - 2026-03-31

- 应用启动完成后会自动执行一次版本检查，发现新版本时给出一次轻提示提醒用户尽快更新。

## [5.2.1] - 2026-03-31

- 修复用户数据迁移时 FFmpeg 工具目录带来的导出、导入和兼容性风险，统一改为导出排除、导入忽略，并提示按需手动拷贝。

## [5.2.2] - 2026-03-31

- 修复用户数据导入后 FFmpeg 手动拷贝提示过度触发的问题；当系统环境变量或当前工具目录已可用时不再误提示。

## [5.2.4] - 2026-04-01

### 发布摘要

- 修复演员头像抓取只保留单一头像链接的问题，改为聚合同名演员的多候选来源。
- 演员头像下载支持多图片 URL 顺序回退，前一个链接失效时会继续尝试后续候选。
- 补齐头像候选持久化与自动化回归，覆盖跨 provider 聚合和头像 URL 失效回退场景。

## [5.3.0] - 2026-04-01

### 发布摘要

- MetaTube 模板库支持配置多个扫描目录，每一行都可直接选择目录，并补齐重复路径、非法路径和离线移动设备路径提示。
- 扫描与抓取阶段会只使用当前可用目录；已保存但暂时不存在的目录会保留配置，并在界面上提示当前不可用。
- 修复新导入影片在 `RefreshMetadataOnly` 场景下虽然已补出演员关系、但演员头像未写入本地缓存的问题。

## [5.4.0] - 2026-04-01

### 发布摘要

- 统一影片卡与合集卡的小、中、大显示规格，所有相关列表页改为同一套响应式卡片密度。
- 设置页补充 `1920x1080` 标准分辨率参考提示，明确三档下大约可展示的卡片数量。
- 新增前端自动化测试，覆盖卡片密度三档参数、归一化逻辑和网格 class 映射。

## [5.5.1] - 2026-04-01

### 发布摘要

- 收口影片批量多选交互，顶部批量工具条只在进入影片多选态后显示。
- 新增“全选当前页影片 / 取消当前页全选”，批量选择范围明确限制为当前页，不跨页累积。
- 合集区继续保持单卡操作，不再引入合集多选，避免与影片级批量动作混淆。
- 媒体库编辑弹窗中的“添加目录”“选择目录”“添加合集目录”已统一改为主题蓝色按钮。
