# 变更日志

本文件面向仓库首页与 public 分发仓库展示，正式版本记录与 [doc/CHANGELOG.md](./doc/CHANGELOG.md) 保持同步。

## [7.0.3]

### 发布摘要

- 修复影视库、收藏页、演员页和标签页在动态分页下的末页回跳问题，翻页、搜索和排序后也会统一回到顶部，不再把上一页的滚动位置带进新页。
- 导入较大用户数据包后再次启动时，Worker 不再每次无条件重跑 taxonomy 全量回填；已有 taxonomy 数据会直接复用，避免在 `Worker ready` 前卡太久触发前端启动超时。
- 为 taxonomy 启动迁移补充“已完成标记 / 现有数据直跳过 / 缺迁移才回填”的决策日志，并新增旧库升级与导入库复用两类回归测试。
- 完成 Worker 全量测试、前端 Vitest 和本地 `7.0.3` 便携包打包，收口本轮 small fixes 的分页稳定性与启动稳定性问题。

## [7.0.1]

### 发布摘要

- 修复媒体库编辑弹窗在多扫描目录场景下的视口溢出问题，目录项较多时不再被直接裁掉，底部动作区也会保持可见。
- 标准库与非标准库现在共用同一套“固定头部 / 固定底部 / 主体区滚动”的弹窗容器，长内容场景下都可以完整浏览和编辑扫描路径。
- 同步补齐相关 UI 规格与 active small fix 文档，归档已完成的 `video-tag-browse` 计划，后续真实使用问题继续统一回到小修复计划承接。

## [7.0.0]

### 发布摘要

- 新增仅面向 `MetaTube 标准库` 的统一标签浏览能力，支持 `类型 / 系列 / 厂商` 三种维度，并可在“全部标准库”与单库范围内查看同类影片。
- 影片详情页头部区重构为新的双栏布局，`类型 / 系列 / 厂商` 标签块支持直接跳转，厂商缺失时会显示局部提示而不是静默留空。
- 后端 taxonomy 数据底座与查询接口已正式收口，旧 `label / 厂牌` 口径退出主链，历史 `Genre / Series / Studio` 数据可直接回填到新的浏览能力。
- 详情页动作区已统一为 `收藏 / 重新抓取 / 播放` 主链，设置页“关于”新增应用内赞助弹窗，并同步补充 README 标签页说明与赞助入口说明。
- 补齐前端标签页、详情页展示与赞助弹窗自动化覆盖，并完成 Worker 全量测试、前端 Vitest、生产构建和 `7.0.0` 便携包回归。

## [6.5.0]

### 发布摘要

- 影片详情页“编辑演员元数据”弹窗中的演员库列表不再固定写死 `按演员名正序`，现在会复用显示设置里的“演员默认排序”。
- 演员页和编辑弹窗已统一收口到共享 `resolveActorSortPreference(...)`，用户改成 `按影片数倒序` 等方案后，两处演员卡顺序会保持一致。
- 弹窗演员查询缓存键已带上排序字段，修改显示设置后重新打开弹窗或重新搜索时，不会继续沿用旧排序结果。
- 新增演员默认排序解析前端测试并完成本地 `6.5.0` 打包回归。

## [6.4.0]

### 发布摘要

- 统一 MetaTube 影片图片内部语义，`poster / thumb / fanart` 改为分别优先使用小图、缩略图和大图来源，减少因图片字段缺项导致的错误 sidecar 写入和反复远端补抓。
- 演员头像失败状态从粗粒度 `no_source` 细分为 `search_no_match`、`resolved_but_no_image`、`invalid_placeholder` 与 `transient_failed`，稳定无头像源的演员不再每轮自动重复扫描。
- active 小修复计划中的“库管理扫描手动停止”和“runtime 日志按分钟命名”已正式收口，并补齐对应自动化验证，不再继续挂在 active plan 里作为未完成项。
- 新增图片 fallback、头像失败状态分流与自动重试拦截回归测试，并完成 Worker 全量测试与前端 Vitest 回归。
- 演员详情页关联影片列表改为复用显示设置中的“影片库默认排序”，不再硬编码 `按 VID 正序`；当用户设置“按发布日期倒序”时，演员作品页也会同步按发布日期倒序打开。

## [6.3.0]

### 发布摘要

- 演员头像抓取新增演员维度状态记忆，自动区分 `resolved / no_source / invalid_placeholder / transient_failed`，避免库级连续补抓时反复重试同一批无头像源演员。
- 头像占位图识别明确只做内容校验，不引入图片指纹；`Now Printing` 这类无效图会被拦截为 `invalid_placeholder`，不再当成真实头像写入缓存。
- 演员列表页与影片详情页的演员卡新增右键动作“重新拉取演员头像”，允许用户对单个演员显式重试，而不是只能整库重抓。
- 修复媒体库统计被旧平铺残留记录和 sidecar 已完整但数据库未回填记录污染的问题，减少 `994 / 990` 这类长期不收敛的统计偏差。

## [6.2.0]

### 发布摘要

- 结果页显示修复从“页面 body 估算 + 固定分页”升级到共享 `ResultGridViewport` 容器，库页、收藏页、演员列表页与演员详情页现在共用同一套 viewport 测量和分页 footer 收口逻辑。
- 影片网格与演员网格统一改为按真实 viewport 高度计算可视行数，并追加轻量 overscan 行，避免继续靠 `固定 4 行` 或页面私有 `body-height reserve` 常量拟合首屏。
- 前后端结果查询新增 `offset / limit` 兼容窗口语义，在保留现有分页交互的前提下，为后续 viewport window 和更自然的长列表浏览体验铺路。
- 补齐前端纯函数与共享组件测试、后端 contract 与窗口查询测试，并完成前端生产构建和 Worker 全量测试回归。

## [6.1.0]

### 发布摘要

- MetaTube 抓取主链继续提速，默认请求预算从 `30 req/min` 提升到 `50 req/min`，保留降级档 `20 req/min + 15s` 启动间隔作为服务端异常时的保护回退。
- 旧库 `missing-only` 抓取现在会优先从可信 `WebType + WebUrl` 提取远端身份直连 detail，并在同一轮任务内按共享 `VID` 复用已解析身份，减少 `search -> detail` 双跳请求。
- `[Library-Scrape]` 日志补齐抓取瓶颈定位字段，新增 `waitReason / requestStage / actorStage` 等口径，后续可以直接区分卡在请求预算、影片并发、演员并发还是降级串行。
- 演员详情页、影片库与收藏页的共用网格继续收口到共享分页/列数实现，并补齐纯函数与轻量组件测试，减少高分辨率场景下的列数与末页展示回归风险。

## [6.0.5]

### 发布摘要

- 继续修复 `1080P / 2K / 4K` 全屏下影片库、收藏页和演员页共用网格的宽度利用问题，列数计算从严格 `floor` 调整为“允许小幅收缩后优先争取多一列”，不再轻易在右侧浪费整列空间。
- 维持“用户卡片密度优先 + 末页不放大”的现有策略，不回退到 `auto-fit + 1fr`；当前页列数一旦确定，会给整页注入统一实际列宽，保证满页和末页卡片宽度一致。
- active roadmap 与正式 `library-page` 规格已补充高分屏全屏验证口径，明确 `1080P / 2K / 4K` 与常见缩放比例下的回归要求。
- 新增 `gridPagination` 纯函数覆盖与共享 `VideoGrid` 轻量组件测试，补齐高分屏典型宽度、多一列命中、整行分页，以及页面样式变量接线不丢失等场景。

## [6.0.4]

### 发布摘要

- 修复影片库、收藏页和演员详情页在分页最后一页条目较少时，`video-grid` 使用 `auto-fit + 1fr` 把剩余卡片横向拉宽的问题；现在会按用户当前选择的影片卡宽度固定列数显示，空位直接留白，不再放大卡片。
- 抓取规划矩阵已正式收口到 `doc/modules/14-scrape-pipeline.md`，明确 `LocalRepair / Repair* / Refresh* / NormalizeFromRemoteDetailOnly / FullScrape` 的命中条件、写回范围和失败回退规则。
- 标准库扫描与非标准本地库扫描的边界、双仓发布链职责与重发口径已同步补进正式模块文档，后续不再依赖 active plan 承载完整规则正文。
- 新增抓取规划与旧库补抓自动化覆盖，补齐可信远端身份、本地多缺项修复、失败后图片 URL 回源与无可恢复工作跳过等回归场景。

## [6.0.3]

### 发布摘要

- 影片库、收藏页和演员详情页的卡片分页不再固定写死 `30` 条，改为按当前 `video-grid` 实际列数自动计算动态 `pageSize`，优先按整行倍数铺满页面。
- 修复 `4K / 高 DPI / 宽窗口` 下因“固定分页大小”和“动态列数”不整除导致的右下角长期空两格、空多格问题，不再继续依赖单纯调整 `auto-fit / auto-fill` 样式碰运气。
- 新增前端 `gridPagination` 计算与自动化测试覆盖，并在库页补充网格分页诊断日志，后续排查卡片密度、列数和分页换算问题更直接。

## [6.0.2]

### 发布摘要

- 细化 MetaTube 抓取失败状态，新增 `failed-search-no-match`、`failed-remote-timeout`、`failed-remote-rate-limit`、`failed-remote-server` 与 `failed-sidecar-write`，失败后排障和筛选不再只靠笼统的 `failed`。
- `[Library-Scrape]` 第一阶段日志补齐了 `libraryId / dataId / vid / plan / scrapeStatus / retryReason / sidecarState / failureStatus` 等关键字段，候选规划、远端回退和失败落库链路更容易直接对齐。
- `publish-public-release.ps1` 新增 public 同版本重发显式开关和发布前预检，默认不再静默覆盖同版本 public release，缺少 `CHANGELOG` 摘要或 ZIP 资产不匹配时会直接中止。
- 同步补充抓取链与双仓发布正式文档，并为新的失败状态分类和兼容重试规则增加自动化测试覆盖。

## [6.0.1]

### 发布摘要

- 演员详情页头部改为更紧凑的超简识别带，只保留 `头像 / 演员名称 / 影片数`，三者在垂直方向上保持居中对齐，作品区能更早进入首屏。
- 演员详情页移除弱信息行和独立 `关联影片` 标题，内容区继续复用库页现有的 `QueryToolbar / video-grid / Pagination` 结构，减少页面间割裂感。
- `doc/UI/new/pages/actor-detail-page.*` 与 `flow/README.md` 已同步更新为新的演员页正式方案，线框、说明文档与实现口径保持一致。

## [6.0.0]

### 发布摘要

- 统一内容抓取链日志前缀为 `[Library-Scrape]`，后续排查 MetaTube 请求、抓取计划、降级节流和 sidecar 写入时不再混用历史前缀。
- 抓取失败但仍缺 `nfo`、海报、缩略图、fanart、演员或演员头像的影片，现在会在下一轮自动重新入队；旧 `NFO` 中失效的 `metatubeid` 命中 `404 / movie not found` 后也会自动回退到搜索刷新链，重新生成标准 `NFO` 并继续补抓缺失内容。
- 影片卡海报 URL 新增基于 sidecar 文件时间戳的 cache bust 参数，前端在 `library.scrape` 与 `video.generate-posters` 任务完成后也会主动失效当前库列表与影片相关查询，减少“文件已写盘但卡片仍显示旧图”的偶发状态。
- 修复 4K / 高 DPI 下库列表最后一行因网格空轨道导致的右下角留白问题，影片、合集和演员网格统一改为 `auto-fit`，现有卡片会自动占满最后一行可用空间。

## [5.9.0]

### 发布摘要

- 调整 MetaTube 抓取节流策略：纯演员补修与本地修复不再沿用远程电影抓取的 `8s` 启动间隔，减少大库补修时的排队时间。
- 演员抓取并发从 `2` 提升到 `4`，让头像补修场景能同时处理更多演员查询。
- 新增扫描任务手动停止能力，扫描进行中按钮会切换为“停止”，允许用户主动中断扫描阶段。
- 补充 `NormalizeFromRemoteDetailOnly` 中间档：旧库 `NFO` 已带 `metatubeid` 时可直接走远端详情标准化，不再先做电影搜索。
- 新增抓取规划、`NFO` 标准化与 `40` 条模拟吞吐测试；在默认参数下，纯演员补修场景的启动排队可从约 `312s` 收敛到 `0s`，演员批次数从 `20` 批降到 `10` 批。

## [5.8.0]

### 发布摘要

- 恢复 MetaTube 标准库扫描时按规范 `VID` 建立影片子目录，普通单文件、分集和已识别影片都会进入对应目录。
- `-C` / `-CH` 中文字幕变体及其副本后缀现在会归入同一个基准 `VID` 目录，继续保持多文件、多卡展示，不做顶替或覆盖。
- sidecar 读写与扫描迁移统一按基准 `VID` 共享 `nfo/poster/thumb/fanart`，并补齐回归测试，确保“标准库扫描后仍平铺”无法再次通过自动化测试。
- 自动抓取规划现在会优先复用本地已持久化 `nfo` / sidecar 快照；已有本地主数据的旧库影片不再误触发 `RefreshMetadataOnly`，只缺演员头像时会直接走演员补修。

## [5.7.2]

### 发布摘要

- 补齐 `AB_12_1`、`SSNI-999(1)`、`MIDE-001 CD2` 与 `ABC-123_uncensored` 的分集识别回归测试，覆盖 5.7.1 未完全落地的样例矩阵。

## [5.7.1]

### 发布摘要

- 修复分集目录跳过判断在一对多索引下退化为每次都不跳过的性能问题，大库重复扫描恢复快速跳过。
- 分集命名正则从非贪婪前缀改为后缀锚定，修复 `FC2-PPV-*_1`、`AB_12_1` 等多段文件名的分集识别漏判。

## [5.7.0]

### 发布摘要

- 标准库扫描只对 `_1`、`(2)`、`CD1`、`PART2` 等高置信度分集命名做同源归目录，不再对同 `VID` 文件执行物理替换。
- 同 `VID` 多文件改为按路径独立入库，并在列表中按一文件一张卡展示，避免多版本、多格式或分集文件被覆盖折叠。
- 扫描阶段明确不再静默删除用户磁盘上的旧视频文件，整体整理策略改为更保守的"只移动分集，不删除文件"。
- 补齐 Worker 自动化测试，覆盖分集归目录、同 `VID` 多记录保留与展示返回口径。

## [5.6.1]

### 发布摘要

- 用户数据导入包体积上限从 1 GB 提升到 5 GB，覆盖演员头像和 sidecar 图片较多的常见重度场景。
- 设置页用户数据导入按钮区域补充 5 GB 限制说明，用户在选中文件后即可被即时拦截，不会等到上传阶段才失败。
- Worker 与前端的导入阈值保持一致，避免只改一侧导致体验或行为不一致。

## [5.6.0]

### 发布摘要

- 用户数据导入改为原子替换，先写 staging 再 swap，中途崩溃或断电可从 backup 自动恢复，杜绝数据丢失。
- 导入/导出期间前端显示不可关闭遮罩，防止用户误操作关闭应用；Worker 启动时自动清理上次残留的导入临时目录。
- 扫描中途如果移动设备目录离线，会自动跳过并在结果摘要中提示，不再产生半扫描状态。
- 前端选择导入包时预校验文件大小，超过 1 GB 限制立即提示用户，避免上传后被服务端拒绝。

## [5.5.2]

- 用户数据导入与导出请求改为更长超时窗口，避免完整库迁移时被前端 30 秒超时提前中断。
- Worker 导入链路补充包路径与大小日志，便于定位大包导入失败阶段。
- 新增稳定的约 `100 MiB` 混合测试包与自动化回归，覆盖头像缓存、预览缓存与 sidecar 缓存导入恢复。
